## v0.3.1 - 2026-04-30
### Added
* Added `skill` subcommand to install, remove, and list the embedded neo4j-cli SKILL.md across detected AI agents (claude-code, cursor, windsurf, copilot, gemini-cli, cline, codex, pi, opencode, junie); the SKILL.md is generated at build time from the cobra command tree so agents always see current usage; `install` writes to `<agent-skills-dir>/neo4j-cli/SKILL.md` (replacing any existing file/symlink), with no agent name installs to every detected agent; `list` is read-only, `install`/`remove` require --rw in agent mode
### Fixed
* Fixed EchoTool.Configure silently discarding the error returned by BaseTool.Configure; the error is now propagated so misconfigured parameters (e.g. non-integer repeat value passed via tool config) are surfaced to the caller instead of leaving the tool in an inconsistent state
* Fixed generated skill file and test seed file being written with world-readable permissions (0644); both gen-skill output and skill service test fixtures now use 0600 so skill files are only readable by the owning user
* Fixed golangci-lint violations across the codebase; unchecked registry.Register error returns in all tool registry tests (TestToolRegistryGet, TestToolRegistryList, TestToolRegistryListNames, TestToolRegistryUnregister, TestToolRegistryExists, TestToolRegistryClear, TestToolRegistryDuplicateRegistration, TestToolRegistryFilter) now use t.Fatalf; unused handler parameters renamed to _ throughout admin, cloud, config, skill, and dispatch command handlers and test fixtures, including PresentationService field in tool.Context and dispatch.Context updated from concrete pointer to presentation.Service interface; concrete service implementation types (ConfigService, LoggerService, PresentationService) made unexported to remove package-name stutter, with NewPresentationService return type narrowed to the Service interface and all external references updated; replaced copy loop in cypher result mapping with copy(); replaced fmt.Sprintf("%s", queryType) with .String(); lowercased capitalised Aura credentials error string per Go convention; replaced WriteString(fmt.Sprintf(...)) with fmt.Fprintf in help tool; removed dead logInfo method from analytics; SkillServiceImpl.List ctx parameter renamed to _; mockAdminService mock methods ctx parameters renamed to _ in admin tests

## v0.3.0 - 2026-04-29
### Added
* Added standard JSON response envelope for all agent-mode output; success responses use {"status":"ok","data":...,"request_id":"...","schema_version":"1"} with data as an object for single-record commands and an array for list commands; error responses use {"status":"error","error":{"code":"...","message":"..."},...}; the two channels (status/data vs status/error) are mutually exclusive and always present
* Added typed CommandResult return value from all command handlers, with separate channels for human rendering (Presentation, FormatOverride) and JSON output (Item, Items, Message); constructors ItemResult, ListResult, and MessageResult enforce consistent population; all command handlers (cypher, cloud, admin, config) updated to return CommandResult
### Fixed
* Fixed cobra-level errors (unknown subcommand, unknown flag) being printed as human-readable text to stderr in agent mode instead of JSON envelopes to stdout; run() now pre-scans os.Args for --agent and --rw before cobra executes so the error handler always sees the correct agent mode state; printAgentError assigns UNKNOWN_COMMAND and UNKNOWN_FLAG error codes to cobra-originated errors so agents can distinguish input mistakes from execution failures
* Fixed --agent flag not activating agent mode when placed after the cypher subcommand; because DisableFlagParsing skips cobra's persistent flag parsing, the agentMode package variable was never set from the command line; dispatchCategory now pre-scans args for --agent and --rw before building the dispatch context and calling isJSONMode, ensuring JSON output and read-only enforcement are active regardless of flag position
* Fixed global persistent flags (--agent, --rw, --timeout, --request-id, and others) being appended to the Cypher query string when placed after the subcommand; the cypher command uses DisableFlagParsing which bypasses cobra's persistent flag parsing, causing all unrecognised tokens to fall through into the query; parseCypherFlags now skips all known global flags via globalBoolFlag and globalValueFlag helpers, and captures --agent and --rw explicitly so they are merged into the dispatch context before query execution

## v0.2.0 - 2026-04-28
### Added
* Added --agent flag and NEO4J_CLI_AGENT=true env var to activate agent-optimised mode: JSON output by default, read-only by default, no interactive prompts, and structured JSON error envelopes on stdout with machine-readable error codes
* Added CypherService.Explain method that pre-pends EXPLAIN to a query and returns the Neo4j query-type classification (r/rw/w/s) from the result summary; used by the agent-mode write-protection check
* Added EXPLAIN-based Cypher write detection: in agent mode without --rw, queries are pre-checked with EXPLAIN and blocked if Neo4j classifies them as write (rw/w/s); EXPLAIN/PROFILE queries run as-is without a pre-check; --rw skips the EXPLAIN round-trip entirely
* Added MutationMode to the Tool interface and dispatch.Command; tools and commands declare ModeRead, ModeWrite, or ModeConditional; the dispatcher enforces the read-only contract in agent mode before Execute is called, with no per-handler special-casing required
* Added --request-id flag and NEO4J_CLI_REQUEST_ID env var for orchestrator-injected correlation IDs; a UUID is auto-generated per invocation when not supplied, included in all agent-mode JSON error envelopes
* Added --rw flag and NEO4J_CLI_RW=true env var to permit write/mutating operations in agent mode; governs Cypher writes, cloud instance mutations (create, update, pause, resume, delete), and destructive config operations (delete, reset) under a single authorization gate
* Added --timeout duration flag (e.g. --timeout 30s) to bound command execution time; expired commands exit with a TIMEOUT agent error on stdout in agent mode
### Changed
* In agent mode, all interactive prompts are suppressed: credential prompts return a structured MISSING_CREDENTIALS error; cloud instances delete and config delete/reset skip confirmation dialogs when --rw is set; the interactive prerequisite variants are only used in human (non-agent) mode
