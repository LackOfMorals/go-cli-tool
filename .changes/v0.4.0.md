## v0.4.0 - 2026-04-30
### Added
* Added analytics startup event emission; EmitStartupEvent is now called on every CLI invocation after the analytics service is initialised and the telemetry preference is applied, recording OS, architecture, CLI version, machine ID, binary path, and uptime to Mixpanel; previously the Analytics infrastructure was fully wired but no events were ever sent
* Added analytics command event emission; a COMMAND_USED event is now sent after every command dispatch carrying the full command path (e.g. "cloud instances list" rather than just "cloud"), a success boolean, and the active agent-relevant flags (agent_mode, allow_writes, output_format, timeout_set); sensitive flags such as credentials and URIs are intentionally excluded; the cypher category never includes query content in the command path; the new EmitCommandEvent method and ActiveFlags type are added to the analytics.Service interface with a corresponding mock, and the previous EmitToolEvent call site in dispatchCategory is replaced
### Removed
* Removed `admin` command and all subcommands (show-users, show-databases); the AdminService interface, AdminServiceImpl, and associated User and Database types have been removed from the service layer; agents requiring database user or database status information should query Neo4j directly via the `cypher` command
* Removed `config` command and all subcommands (list, set, delete, reset); CLI configuration is now sourced exclusively from environment variables (CLI_ prefix) and persistent flags; the --config-file flag has been removed; SaveConfiguration has been removed from the config.Service interface; config file reading and writing has been removed from the config package while Viper is retained for env var binding
* Removed interactive credential prompts (InteractiveNeo4jPrerequisite, InteractiveAuraPrerequisite) that previously prompted for connection details and saved them to the config file; Neo4j and Aura credentials must now be supplied via environment variables (CLI_NEO4J_URI, CLI_NEO4J_USERNAME, CLI_NEO4J_PASSWORD, CLI_AURA_CLIENT_ID, CLI_AURA_CLIENT_SECRET) or persistent flags; the non-interactive Neo4jPrerequisite and AuraPrerequisite checks remain and error messages have been updated to reference env vars
* Removed the cobra-generated `completion` command (shell tab-completion for bash, zsh, fish, powershell); the command is not useful for the primary agent audience and was suppressed via CompletionOptions.DisableDefaultCmd
